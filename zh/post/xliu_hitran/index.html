HITRAN 数据库用法

刘训臣

十一月 8, 2025

引言

在上一篇文章 吸收光谱简介 中，我们介绍了分子吸收光谱的基本原理。实际应用中，我们需要借助光谱数据库（如 HITRAN）和数值模型来仿真分子的吸收特征。

常见的分子光谱的数据库有：

  • HITRAN, USA Harvard-Smithsonian Center for Astrophysics Molecular Data
  • NASA JPL Molecular Spectroscopy, USA
  • The Cologne Database for Molecular Spectroscopy, CDMS, Germany
  • GEISA Spectroscopic Database, France
  • Pacific Northwest National Labs Spectral Database, USA
  • Ames Research Center Infrared Spectral Database of PAH, USA
  • MPI-Mainz-UV-VIS Spectral Atlas of Gaseous Molecules, Germany
  • surface types: US Geological Society Digital Spectral Laboratory, USA
  • NIST Chemistry WebBook, USA
  • AIST Integrated Spectral Data Base System for Organic Compounds, Japan
  • Bremen, Institute of Environmental Physics, Germany
  • LISA database (SCOOP), France
  • Institute of Atmospheric Optics (CO2 O3), Tomsk, Russia
  • Instituto Superior Tecnico SPATRAN, Gas and Plasma Radiation Database, Italy

HITRAN数据库是我们最常用的分子光谱谱线的数据库。NASA JPL 数据库中有很多星际分子的谱线，另外JPL数据库用的 SPCAT/SPFIT 软件也是功能非常强大。未来我们会尝试用 SPCAT 模拟高温反应流体的分子光谱。Tomsk 有很好的CO2的光谱数据库。

模拟代码

用HITRAN模拟一个吸收峰，我们参考 https://hitran.org/docs/definitions-and-units/ 中的定义。 需要光谱模型中的三个参数：

1）谱线位置，这需要吸收峰对应的上下能级的能级差，数据库给出了谱线位置和下能级，因此可以计算上能级位置；

2）谱线强度，定义如上篇文章给出 网页中公式4，HITRAN 数据库给出了参考温度下乘了同位素丰度的线强$S$，单位为$[\mathrm{cm^2 \cdot cm^{-1}}]$。

3）谱线展宽的线形，定义为Voigt展宽的线形，数据库给出了压力展宽的温度系数和展宽吸收

对于 每行160位的数据，可以用下面 python 代码读取每个跃迁的参数。

# HITRAN 160位格式读取函数
def read_hitran_par(filename):
    """
    读取HITRAN 160位格式的par文件
    返回格式: [波数, 线强, 爱因斯坦A系数, 空气加宽半宽, 自加宽半宽, 低态能量, 温度依赖系数, 压力位移]
    """
    database = []
    
    with open(filename, 'r') as f:
        for line_num, line in enumerate(f, 1):
            line = line.rstrip('\n')  # 只去掉行尾换行符，保留其他空白
            if len(line) < 160:  # 确保行长度足够
                print(f"警告: 第 {line_num} 行长度不足160字符，已跳过")
                continue
                
            try:
                # 根据HITRAN格式定义解析各个字段
                nu = float(line[3:15])           # 波数
                S = float(line[15:25])           # 线强
                A = float(line[25:35])           # 爱因斯坦A系数
                gamma_air = float(line[35:40])   # 空气加宽半宽
                gamma_self = float(line[40:45])  # 自加宽半宽
                E = float(line[45:55])           # 低态能量
                n_air = float(line[55:59])       # 温度依赖系数
                delta_air = float(line[59:67])   # 压力位移
                
                # 将解析的数据添加到数据库
                database.append([nu, S, A, gamma_air, gamma_self, E, n_air, delta_air])
                
            except ValueError as e:
                print(f"解析第 {line_num} 行时出错: {repr(line)}")
                print(f"错误信息: {e}")
                continue
    
    print(f"成功读取 {len(database)} 条谱线")
    return np.array(database)

线宽模型的计算我们用Faddeeva 函数计算Voigt 线形。 Voigt 线型是Gaussian展宽（温度导致的速度展宽）和压力展宽的卷积，其中温度展宽的计算需要用到不同的分子同位素的质量。

压力展宽用的是数据库中的参数。

这里可以忽略 gamma_self 自展宽，复杂环境中各种分子的碰撞展宽太多了，就不考虑自展宽了。这样Voigt 线形可以只计算一次，大大节约计算时间。

注意这里用的是 CGS 单位制的物理常数。


# physical constants

T_ref = 296.0
cBolts = constants.Boltzmann*1E7 # CGS Boltzmann constant unit is erg/K, CGS, c is cm/s 
cc = constants.speed_of_light*1E2 # CGS
cNA = constants.N_A
c2 = constants.physical_constants['second radiation constant'][0]*100 # CGS cm K
cP = constants.physical_constants['standard atmosphere'][0]*10 # pressure unit in CGS is Ba. covert to 0.1 Pa 

##################################################################################
# Voigt lineshape from Faddeeva function

cGammaD = math.sqrt(2*cBolts*cNA*math.log(2))/cc
def profile_voigt(Mass,nu,gamma_air,n_air,delta_air,T,p,wavenumber): # return voigt profile on wavenumber, function of (T,p)
    gamma = gamma_air*p*((T_ref/T)**n_air)#+gamma_self*p*c*((T_ref/T)**n_self) # Eqn 6
    #gamma = gamma_air*gamma_air_ratio*p*(1-c)*((T_ref/T)**n_air)+gamma_self*p*c*((T_ref/T)**n_self) # Eqn 6
    GammaD = cGammaD*math.sqrt(T/Mass)*nu# + laser # eqn (5), 
    sigma = GammaD/(math.sqrt(2*math.log(2)))
    variable = (wavenumber- nu - delta_air*p+gamma*1j)/(sigma*math.sqrt(2)) # air shift
    voigt = (special.wofz(variable)).real/(sigma*math.sqrt(2*constants.pi))
    return voigt

接下来只需要数据库par 文件就可以一行一行的叠加计算了。 计算调用coef 吸收系数，OD 为乘了分子密度和吸收距离的光学厚度。coef 和 OD 可以线性叠加。

##################################################################################
# line-by-line coef calculation
def coef(Mass,T,p,database,partition_function,wavenumber):#,color): # k  absorption coef 
    coef = np.zeros(len(wavenumber))
    for line in database:
        nu          = line[0]
        S           = line[1]
        gamma_air   = line[3]
        E           = line[5]
        n_air       = line[6]
        delta_air   = line[7]
        profile = profile_voigt(Mass,nu,gamma_air,n_air,delta_air,T,p,wavenumber)
        intensity = S * partition_function[int(T_ref-1)]/partition_function[int(T-1)] * math.exp(-c2*E/T)/math.exp(-c2*E/T_ref)*(1-math.exp(-c2*nu/T))/(1-math.exp(-c2*nu/T_ref)) # Eqn 4, T is rounded
        #        ax1.bar(nu,peak,align='center',width=0.001,color='b')
        coef += profile*intensity
    return coef

def OD(Mass,T,p,c,l,database,partition_function,wavenumber): # optical density # k*[X]*l   absorption coef times volume density, assuming total pressure p and c is pressure fraction. c needs to times length in unit cm to get dimensionless optical depth \tao, bkg is added at the end
    acoef = coef(Mass,T,p,database,partition_function,wavenumber)
    density = p*c*cP/(cBolts) /T # volumn density /cm^3 from p/kT  # check if concentration change
    OD = acoef*density*l
    Tr = np.exp(-OD)
    Ab = 1-Tr
    return OD,Ab,Tr

1. 目录结构与模块划分

从单分子工具升级为多分子混合光谱仿真系统，目录结构如下：

flame_spectrum/
├── core/                          # 核心计算引擎
│   └── hitran_spectrum.py         # 多分子 HITRAN 引擎（谱线解析、Voigt 线型、混合光谱）
├── gui/                           # 可复用 GUI 组件
│   ├── components.py              # 通用后台计算线程（CalculationThread）
│   └── spectrum_plotter.py        # 高级光谱绘图画布（双轴、双 X 轴、智能下采样）
├── apps/                          # 应用程序入口
│   └── hitran_gui.py              # 多分子混合光谱仿真图形界面
├── hitran_database/               # 谱线与配分函数数据
│   ├── 01_H2O/, 02_CO2/, ...      # 各分子 .par 文件
│   └── Q/                         # 配分函数文件（q5.txt, q1.txt 等）
└── README.md

各模块职责

  • core/hitran_spectrum.py
    封装了 HITRAN 160 位格式解析、分子与同位素自动识别（内置 ISO 字典）、配分函数线性插值、多普勒标准差直接计算、Voigt 线型、逐线吸收截面计算以及多分子混合光谱。用户通过 add_molecule() 添加任意数量分子并指定各自浓度，调用 OD_mixture() 即可获得总透射率、总吸收系数及各分子分量。

  • gui/components.py
    提供通用后台计算线程 CalculationThread，将耗时计算移出主线程，避免界面冻结。线程直接调用 HitranSpectrum.OD_mixture，完成后通过信号返回结果字典，与界面完全解耦。

  • gui/spectrum_plotter.py
    提供 SpectrumCanvas 类（基于 FigureCanvasQTAgg），内部集成 Matplotlib 画布和导航工具栏。专门实现了 plot_mixture() 方法，用于绘制混合光谱：总吸收系数（红色实线）、各分子分量（彩色虚线）、透射率（蓝色实线，右轴）以及波数/波长双 X 轴，并支持自适应下采样和可调网格密度。

  • apps/hitran_gui.py
    多分子光谱仿真图形界面主程序。左侧控制面板支持多 .par 文件添加/移除、每分子浓度独立设置（ppm）、配分函数文件夹选择、温度/压力/光程/omega_wing 等全局参数调节以及波数范围设定。点击计算后，后台线程启动，结果由 SpectrumCanvas 直接呈现，界面响应流畅。

这种结构将核心计算、后台线程、专业绘图与用户界面彻底分离，任一模块均可独立测试或复用到其他项目（如火焰光谱主程序、传感器仿真工具等）。

2. 核心引擎：hitran_spectrum.py（多分子版）

自动识别分子与同位素

传统的做法需要用户手动指定分子 ID 和同位素 ID，再去找对应的配分函数文件。我们的新程序能够自动完成这一过程：

  • 读取 .par 文件时，解析每行的第 1–2 列（分子 ID）和第 3 列（同位素 ID）。
  • 利用内置的 ISO 字典（涵盖 HITRAN 所有分子及其同位素），自动匹配出分子名称、分子质量、自然丰度等信息。
  • 根据分子 ID 自动在 Q/ 文件夹中寻找对应的配分函数文件（如 q5.txt 对应 CO），无需用户干预。
# 自动识别示例
hitran = HitranSpectrum()
hitran.load_data('CO_1416.par', 'Q/')
# 此时 hitran.molecule_info 已包含分子名称、ID、质量等

配分函数的自动匹配与插值

配分函数是计算温度依赖线强的关键。 配分函数从 Q/q{ID}.txt 读取温度‑Q值两列，计算时使用 线性插值（np.interp）得到任意温度下的 Q，保证线强修正精度。

波数范围的灵活设置

程序会自动给出 .par 文件中包含的波数范围（default_start 和 default_end），用户也可以手动指定起始和结束波数，以及分辨率。这为聚焦于特定谱带提供了便利。

谱线计算域倍数 omega_wing

参考 HAPI（HITRAN Application Programming Interface）的定义，我们引入了 omega_wing 参数。它控制每条谱线的计算域宽度：

每条谱线只在中心频率 ± omega_wing × (多普勒展宽 + 压力展宽) 的范围内计算，远处的谱线贡献忽略不计。

这在大波数范围（例如 4000–5000 cm⁻¹）计算时能大幅节省时间，因为每条谱线只需在其局部范围内积分，而不是在整个波数网格上逐点计算。默认值 omega_wing = 10 在精度和速度之间取得了良好平衡。

主要方法

  • coef(T, p, c, wavenumber=None, start=None, end=None, resolution=0.001, omega_wing=10)：返回吸收系数数组和对应的波数网格。
  • OD(T, p, c, l, ...)：计算光学深度、透射率和吸收率，并返回波数和吸收系数。

关键计算修正

  • 多普勒标准差直接由物理推导给出，无需半宽转换：
    σ = (ν / c_light) * sqrt(k_B T / m)，其中 m = M_mol / N_A（单分子质量）。
  • 压力加宽忽略自加宽，仅依赖总压 p，便于混合气体计算。
  • 吸收截面与浓度分离：单个分子的 cross_section() 返回 σ(ν) [cm²/molecule]，混合时根据各分子体积分数分别计算数密度 N_i = p·c_i/(k_B T)，总吸收系数 k_total = Σ N_i·σ_i，避免浓度单位错误。

3. 通用绘图组件：spectrum_plotter.py

为了在多个应用中复用绘图逻辑，我们设计了独立的 SpectrumCanvas 类。它基于 FigureCanvasQTAgg，并集成了以下特性：

  • 双 Y 轴：左轴显示吸收系数（蓝色），右轴显示透射率（橙色），颜色区分清晰。
  • 双 X 轴：底部为波数（cm⁻¹），顶部自动换算为波长（µm），方便从不同物理尺度观察。
  • 智能下采样：当数据点数超过 50 万时自动降采样，避免界面卡顿。
  • 网格密度可调：提供 fine（精细）、medium（中等）、coarse（稀疏）三种选项。
  • 标题信息：自动在图形标题中嵌入温度、压力、光程等参数。

该组件与业务逻辑完全解耦，可被任意需要绘制光谱的应用程序调用，例如后续的火焰光谱主程序。

4. 图形界面：hitran_gui.py

我们为单分子光谱仿真制作了一个完整的图形界面，操作直观：

使用步骤

  1. 选择文件：点击“浏览”分别选择 .par 谱线文件和配分函数文件夹（Q/）。
  2. 识别分子：点击“加载数据”，程序自动识别分子并显示其名称、ID、同位素和质量。

5. 补充技巧：编辑 .par 文件

很多时候，我们并不需要计算整个 .par 文件中的所有谱线，而只关心某个同位素或某个谱带。例如，CO 的 .par 文件可能包含多个同位素（¹²C¹⁶O、¹³C¹⁶O 等），而我们只想模拟 ¹²C¹⁶O 在 2000–2200 cm⁻¹ 范围内的吸收。

这时，可以直接用文本编辑工具（如 vi、emacs 或 sed）对 .par 文件进行筛选：

  • 按分子 ID 筛选：每行的前 2 个字符代表分子 ID，例如 05 对应 CO，02 对应 CO₂。
  • 按波数范围筛选：波数位于第 4–15 列（12 个字符），可以按数值范围删减行。

虽然 HAPI 等库提供了基于 Python 的筛选接口，但实际使用中我们发现，直接编辑 .par 文件更为直接高效，尤其是处理大规模数据时。因此，我们的工具并未内置筛选功能，而是建议用户根据需求自行编辑文件。这也保持了工具的简洁性和通用性。

  • apps/hitran_gui.py：完整的图形界面应用程序入口，组装上述模块，提供文件选择、多分子浓度设置、参数调节、计算和绘图功能。

这种分层结构使得核心计算与界面逻辑完全分离，既便于维护，也利于在后续项目（如火焰光谱模拟、传感器设计）中直接复用 core 和 gui 模块。

2. 核心引擎：hitran_spectrum.py（多分子版）

自动识别分子与同位素

  • 读取 .par 文件时，自动解析每行的分子 ID（第 1‑2 列）和同位素 ID（第 3 列）。
  • 利用内置的 ISO 字典（涵盖 HITRAN 所有分子及其同位素），自动匹配出分子名称（如 CO、H2O）、同位素组成（如 (12C)(16O)）、单分子质量（g/mol）、天然丰度等信息。
  • 根据分子 ID 自动在 Q/ 文件夹中匹配对应的配分函数文件（如 q5.txt 对应 CO，q1.txt 对应 H₂O），无需用户干预。
hitran = HitranSpectrum(q_folder='Q/')
hitran.add_molecule('CO_1416.par', concentration=0.01, name='CO')
# 自动读取 Q/q5.txt，获取质量 27.9949 g/mol，识别为 CO

配分函数的自动匹配与插值

配分函数是计算温度依赖线强的关键。程序从 Q/q{ID}.txt 读取温度‑Q值两列，在计算时使用 线性插值（np.interp）获得任意温度下的配分函数值，废弃了早期版本中“温度取整作为数组下标”的错误做法，确保了线强修正的准确性。

波数范围灵活设置与翼截断

  • 程序会自动给出每个 .par 文件包含的波数范围（default_start 和 default_end），用户也可手动指定起始波数、结束波数和分辨率，便于聚焦特定谱带。
  • 引入 omega_wing 参数（默认值 10），参考 HAPI（HITRAN Application Programming Interface）的定义：每条谱线只在中心波数 ± omega_wing×(多普勒半宽 + 压力半宽) 的范围内计算，远处谱线的贡献忽略不计。这比固定波数截断更符合物理，大幅节省宽波段计算时间。

线型计算与浓度处理（关键修正）

  • 多普勒标准差直接由物理推导给出，无需半宽转换： $$\sigma = \frac{\nu}{c} \sqrt{\frac{k_B T}{m}}$$， 其中 $m = M_{\text{mol}}/N_A$（单分子质量）。
  • 压力加宽忽略自加宽项，仅依赖总压 p：
    $$\gamma(p,T) = \gamma_{\text{air}} \cdot (p/p_{\text{ref}}) \cdot (T_{\text{ref}}/T)^{n_{\text{air}}}$$ 便于混合气体计算。
  • 吸收截面与数密度分离：cross_section() 返回单个分子的吸收截面 $\sigma(\nu)$ [cm²/molecule]，不含浓度信息。混合气体总吸收系数由各分子数密度$N_i = p \cdot c_i / (k_B T)$ 与截面相乘后求和得到 $k_{\text{total}} = \sum N_i \cdot \sigma_i$，逻辑清晰，避免了早期版本中浓度单位混淆的问题。
  • 线强处理：HITRAN 数据库中的线强 S 已包含天然同位素丰度，因此使用总分子浓度直接乘以 S 即可获得正确吸收，无需手动再次矫正。

多分子混合接口示例

hitran = HitranSpectrum(q_folder='Q/')
hitran.add_molecule('CO_1416.par', concentration=0.01, name='CO')
hitran.add_molecule('HITRAN_2073-2074.par', concentration=0.02, name='other')

T, p, L = 600.0, 1.0, 10.0
OD, Ab, Tr, wn, total_k, ind_k = hitran.OD_mixture(
    T, p, L, start=2000, end=2200, resolution=0.01, wing=10
)
# ind_k: {'CO': array, 'other': array} ; total_k: 混合总吸收系数

3. 通用绘图组件：spectrum_plotter.py

SpectrumCanvas 类基于 FigureCanvasQTAgg，集成了以下可复用特性：

  • 双 Y 轴：左轴显示吸收系数（总及各分子分量），右轴显示透射率，颜色区分清晰。
  • 双 X 轴：底部为波数（cm⁻¹），顶部自动换算为波长（µm），方便从不同物理尺度观察。
  • 智能下采样：当数据点数超过 50 万时自动降采样，避免界面卡顿。
  • 网格密度可调：提供 fine（精细）、medium（中等）、coarse（稀疏）三种选项。
  • 标题信息：自动在图形标题中嵌入温度、压力、光程等参数。

该组件与业务逻辑完全解耦，可被任意需要绘制光谱的应用程序调用。

4. 图形界面：hitran_gui.py

  1. 配分函数文件夹：指向包含所有 q*.txt 的目录（如 Q/），界面启动时自动匹配默认路径。
  2. 添加 par 文件：可多选文件，每行显示文件名和默认浓度（优先从 Cantera 组分推断，否则设为 10 ppm）。同时自动更新全局波数范围。
  3. 浓度编辑：双击表格浓度单元格直接修改，或选中多行后通过“设置浓度”按钮批量赋值。
  4. 全局参数：温度 (100‑5000 K)、压力 (0.001‑100 atm)、光程 (0.1‑10000 cm)、omega_wing (0.1‑500 cm⁻¹)、波数范围与分辨率（0.0001‑1.0 cm⁻¹）均可通过双精度微调框灵活设定。
  5. 开始计算：点击按钮后，后台 CalculationThread 执行多分子混合光谱计算。完成后右侧绘图区显示：
    • 透射率（蓝色实线，右轴）
    • 总吸收系数（红色实线）及各分子贡献（彩色虚线，左轴）
    • 顶部波长轴（µm），自动同步波数轴范围
  6. 结果统计：下方信息框显示温度、压力、光程及所有已加载分子的名称列表。
  7. 清除图形：一键清空画布和统计信息，方便重复实验。

所有计算均在后台线程中进行，界面不会卡顿。程序入口位于 apps/hitran_gui.py，直接运行即可启动。

5. 补充技巧：编辑 .par 文件

很多时候，我们并不需要计算整个 .par 文件中的所有谱线，而只关心某个同位素或某个谱带。例如，CO 的 .par 文件可能包含多个同位素（¹²C¹⁶O、¹³C¹⁶O 等），而我们只想模拟 ¹²C¹⁶O 在 2000–2200 cm⁻¹ 范围内的吸收。

这时，可以直接用文本编辑工具（如 vi、emacs 或 sed）对 .par 文件进行筛选：

  • 按分子 ID 筛选：每行的前 2 个字符代表分子 ID，例如 05 对应 CO，02 对应 CO₂。
  • 按波数范围筛选：波数位于第 4–15 列（12 个字符），可以按数值范围删减行。

虽然 HAPI 等库提供了基于 Python 的筛选接口，但实际使用中我们发现，直接编辑 .par 文件更为直接高效，尤其是处理大规模数据时。因此，我们的工具并未内置筛选功能，而是建议用户根据需求自行编辑文件。这也保持了工具的简洁性和通用性。

6. 下载与使用

所有重构后的代码和示例数据（hitran_database/）已整合到项目仓库：

  • 核心引擎：hitran_spectrum.py
  • 绘图组件：spectrum_plotter.py 和 components.py

  • 示例数据：hitran_database/（包含各分子 .par 文件和 Q/ 配分函数文件夹）

cd /path/to/flame_spectrum
python apps/hitran_gui.py          # 启动图形界面
python core/hitran_spectrum.py     # 命令行多分子示例
