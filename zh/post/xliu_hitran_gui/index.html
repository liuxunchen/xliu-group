HITRAN 仿真图形界面

图形界面与多分子扩展

1. 目录结构与模块划分

从单分子工具升级为多分子混合光谱仿真系统，目录结构如下：

flame_spectrum/
├── core/                          # 核心计算引擎
│   └── hitran_spectrum.py         # 多分子 HITRAN 引擎（谱线解析、Voigt 线型、混合光谱）
├── gui/                           # 可复用 GUI 组件
│   ├── components.py              # 通用后台计算线程（CalculationThread）
│   └── spectrum_plotter.py        # 高级光谱绘图画布（双轴、双 X 轴、智能下采样）
├── apps/                          # 应用程序入口
│   └── hitran_gui.py              # 多分子混合光谱仿真图形界面
├── hitran_database/               # 谱线与配分函数数据
│   ├── 01_H2O/, 02_CO2/, ...      # 各分子 .par 文件
│   └── Q/                         # 配分函数文件（q5.txt, q1.txt 等）
└── README.md

各模块职责

  • core/hitran_spectrum.py
    封装了 HITRAN 160 位格式解析、分子与同位素自动识别（内置 ISO 字典）、配分函数线性插值、多普勒标准差直接计算、Voigt 线型、逐线吸收截面计算以及多分子混合光谱。用户通过 add_molecule() 添加任意数量分子并指定各自浓度，调用 OD_mixture() 即可获得总透射率、总吸收系数及各分子分量。

  • gui/components.py
    提供通用后台计算线程 CalculationThread，将耗时计算移出主线程，避免界面冻结。线程直接调用 HitranSpectrum.OD_mixture，完成后通过信号返回结果字典，与界面完全解耦。

  • gui/spectrum_plotter.py
    提供 SpectrumCanvas 类（基于 FigureCanvasQTAgg），内部集成 Matplotlib 画布和导航工具栏。专门实现了 plot_mixture() 方法，用于绘制混合光谱：总吸收系数（红色实线）、各分子分量（彩色虚线）、透射率（蓝色实线，右轴）以及波数/波长双 X 轴，并支持自适应下采样和可调网格密度。

  • apps/hitran_gui.py
    多分子光谱仿真图形界面主程序。左侧控制面板支持多 .par 文件添加/移除、每分子浓度独立设置（ppm）、配分函数文件夹选择、温度/压力/光程/omega_wing 等全局参数调节以及波数范围设定。点击计算后，后台线程启动，结果由 SpectrumCanvas 直接呈现，界面响应流畅。

这种结构将核心计算、后台线程、专业绘图与用户界面彻底分离，任一模块均可独立测试或复用到其他项目（如火焰光谱主程序、传感器仿真工具等）。

2. 核心引擎：hitran_spectrum.py（多分子版）

自动识别分子与同位素

传统的做法需要用户手动指定分子 ID 和同位素 ID，再去找对应的配分函数文件。我们的新程序能够自动完成这一过程：

  • 读取 .par 文件时，解析每行的第 1–2 列（分子 ID）和第 3 列（同位素 ID）。
  • 利用内置的 ISO 字典（涵盖 HITRAN 所有分子及其同位素），自动匹配出分子名称、分子质量、自然丰度等信息。
  • 根据分子 ID 自动在 Q/ 文件夹中寻找对应的配分函数文件（如 q5.txt 对应 CO），无需用户干预。
# 自动识别示例
hitran = HitranSpectrum()
hitran.load_data('CO_1416.par', 'Q/')
# 此时 hitran.molecule_info 已包含分子名称、ID、质量等

配分函数的自动匹配与插值

配分函数是计算温度依赖线强的关键。 配分函数从 Q/q{ID}.txt 读取温度‑Q值两列，计算时使用 线性插值（np.interp）得到任意温度下的 Q，保证线强修正精度。

波数范围的灵活设置

程序会自动给出 .par 文件中包含的波数范围（default_start 和 default_end），用户也可以手动指定起始和结束波数，以及分辨率。这为聚焦于特定谱带提供了便利。

谱线计算域倍数 omega_wing

参考 HAPI（HITRAN Application Programming Interface）的定义，我们引入了 omega_wing 参数。它控制每条谱线的计算域宽度：

每条谱线只在中心频率 ± omega_wing × (多普勒展宽 + 压力展宽) 的范围内计算，远处的谱线贡献忽略不计。

这在大波数范围（例如 4000–5000 cm⁻¹）计算时能大幅节省时间，因为每条谱线只需在其局部范围内积分，而不是在整个波数网格上逐点计算。默认值 omega_wing = 10 在精度和速度之间取得了良好平衡。

主要方法

  • coef(T, p, c, wavenumber=None, start=None, end=None, resolution=0.001, omega_wing=10)：返回吸收系数数组和对应的波数网格。
  • OD(T, p, c, l, ...)：计算光学深度、透射率和吸收率，并返回波数和吸收系数。

关键计算修正

  • 多普勒标准差直接由物理推导给出，无需半宽转换：
    σ = (ν / c_light) * sqrt(k_B T / m)，其中 m = M_mol / N_A（单分子质量）。
  • 压力加宽忽略自加宽，仅依赖总压 p，便于混合气体计算。
  • 吸收截面与浓度分离：单个分子的 cross_section() 返回 σ(ν) [cm²/molecule]，混合时根据各分子体积分数分别计算数密度 N_i = p·c_i/(k_B T)，总吸收系数 k_total = Σ N_i·σ_i，避免浓度单位错误。

多分子接口示例

hitran = HitranSpectrum(q_folder='Q/')
hitran.add_molecule('CO_1416.par', concentration=0.01, name='CO')
hitran.add_molecule('H2O.par', concentration=0.02, name='H2O')

T, p, L = 600.0, 1.0, 10.0
OD, Ab, Tr, wn, total_k, ind_k = hitran.OD_mixture(
    T, p, L, start=2000, end=2200, resolution=0.01, wing=10
)
# ind_k: {'CO': array, 'H2O': array} ; total_k: 混合总吸收系数

3. 图形界面：hitran_gui.py

界面已重构为多分子版本，操作流程：

  1. 配分函数文件夹：指向包含所有 q*.txt 的目录（如 Q/）。
  2. 添加 par 文件：可多选文件，每行显示文件名和浓度（默认从 Cantera 组分推断或 10 ppm）。
  3. 浓度编辑：双击表格浓度单元格直接修改，或选中行后通过按钮批量设置。
  4. 全局参数：温度、压力、光程、omega_wing、波数范围与分辨率。
  5. 开始计算：后台线程运行，右侧绘图区显示：
    • 透射率（蓝线，左轴）
    • 总吸收系数（红虚线）及各分子贡献（不同颜色虚线，右轴）
    • 顶部波长轴（µm）
  6. 结果统计：显示温度、压力、光程及所有分子名称。

所有计算在后台进行，界面无卡顿。程序入口位于 apps/hitran_gui.py，直接运行即可启动。

4. 运行环境与启动方式

cd flame_spectrum
python apps/hitran_gui.py          # 启动图形界面
python core/hitran_spectrum.py     # 命令行多分子示例

依赖：Python ≥3.9，numpy, scipy, matplotlib, PyQt6。

3. 通用绘图组件：spectrum_plotter.py

为了在多个应用中复用绘图逻辑，我们设计了独立的 SpectrumCanvas 类。它基于 FigureCanvasQTAgg，并集成了以下特性：

  • 双 Y 轴：左轴显示吸收系数（蓝色），右轴显示透射率（橙色），颜色区分清晰。
  • 双 X 轴：底部为波数（cm⁻¹），顶部自动换算为波长（µm），方便从不同物理尺度观察。
  • 智能下采样：当数据点数超过 50 万时自动降采样，避免界面卡顿。
  • 网格密度可调：提供 fine（精细）、medium（中等）、coarse（稀疏）三种选项。
  • 标题信息：自动在图形标题中嵌入温度、压力、光程等参数。

该组件与业务逻辑完全解耦，可被任意需要绘制光谱的应用程序调用，例如后续的火焰光谱主程序。

4. 图形界面：hitran_gui.py

我们为单分子光谱仿真制作了一个完整的图形界面，操作直观：

使用步骤

  1. 选择文件：点击“浏览”分别选择 .par 谱线文件和配分函数文件夹（Q/）。
  2. 识别分子：点击“加载数据”，程序自动识别分子并显示其名称、ID、同位素和质量。
  3. 设置参数：输入温度 (K)、压力 (atm)、浓度 (ppm)、光程 (cm)。
  4. 波数范围：可勾选“使用数据库默认范围”，或手动设置起始、结束波数和分辨率。
  5. 计算：点击“开始计算”，后台线程执行谱线累加，完成后右侧图形区显示吸收系数和透射率曲线，左侧统计区给出最大值、最小值、平均值等数据。
  6. 保存数据：点击“保存吸收系数”可将波数和吸收系数导出为文本文件，文件头包含完整的元数据。

所有计算均在后台线程中进行，界面不会卡顿。

5. 补充技巧：编辑 .par 文件

很多时候，我们并不需要计算整个 .par 文件中的所有谱线，而只关心某个同位素或某个谱带。例如，CO 的 .par 文件可能包含多个同位素（¹²C¹⁶O、¹³C¹⁶O 等），而我们只想模拟 ¹²C¹⁶O 在 2000–2200 cm⁻¹ 范围内的吸收。

这时，可以直接用文本编辑工具（如 vi、emacs 或 sed）对 .par 文件进行筛选：

  • 按分子 ID 筛选：每行的前 2 个字符代表分子 ID，例如 05 对应 CO，02 对应 CO₂。
  • 按波数范围筛选：波数位于第 4–15 列（12 个字符），可以按数值范围删减行。

虽然 HAPI 等库提供了基于 Python 的筛选接口，但实际使用中我们发现，直接编辑 .par 文件更为直接高效，尤其是处理大规模数据时。因此，我们的工具并未内置筛选功能，而是建议用户根据需求自行编辑文件。这也保持了工具的简洁性和通用性。

6. 下载与使用

所有重构后的代码和示例数据（hitran_database/）已整合到项目目录中。 flame spctrum 文件夹提供完整的更新后的代码包和数据库文件下载：

  • 核心引擎：hitran_spectrum.py
  • 绘图组件：spectrum_plotter.py 和 components.py
  • 图形界面：hitran_gui.py
  • 示例数据：hitran_database/（包含各分子 .par 文件和 Q/ 配分函数文件夹）

运行环境：Python 3.9+，需安装 numpy, scipy, matplotlib, PyQt6（或 PyQt5）。启动方式：

cd /path/to/flame_spectrum
python apps/hitran_gui.py

后记

将原本的函数式脚本升级为模块化、可复用的工具集。核心算法封装于 core，绘图组件独立于 gui，应用入口清晰简洁。这不仅提升了代码质量，也为后续集成化学平衡计算（Cantera）和多分子混合光谱（hitran_spectrum_dual.py）铺平了道路。

未来，我们计划进一步优化计算性能（如并行化谱线循环），并扩展支持其他数据库（如 JPL、GEISA）。欢迎读者提出建议或贡献代码！
