PIV/PLIF Liutex 分析工具 – 基于 PyQt6 的流场涡识别与频谱分析软件

引言

在实验流体力学中，粒子图像测速（PIV） 与 平面激光诱导荧光（PLIF） 是两种广泛使用的光学测量技术。前者能获得瞬时速度场，后者可捕捉标量（如浓度、温度）的空间分布。如何从大量实验数据中高效提取有物理意义的信息，尤其是分离出真实的涡结构，一直是个挑战。

近年来，Liutex 涡识别方法提供了一种严格、无阈值的旋转分解，能够从速度梯度张量中剥离出刚性旋转强度，避免传统判据（如 Q、λ₂）将剪切误判为涡的缺点。

Liutex 方法简介

我们基于此理论，开发了一款跨平台、模块化的 GUI 软件，集成 PIV/PLIF 数据加载、Liutex 计算、多物理量可视化、动画播放以及单点频谱分析等功能。

软件主要功能

1. 多格式数据加载

  • PIV：支持 DaVis 输出的标准 .txt 文件（包括 #DaVis 头格式）以及 TECPLOT 风格的 .dat 文件。自动识别格式，无需手动指定。
  • PLIF：读取 TIFF 图像序列，并提供边缘保持滤波、阈值掩膜、模糊增强等预处理选项，便于与速度场叠加显示。
  • Bin 平均：可自定义下采样倍数，降低数据量并过滤高频噪声。
  • 原点平移：支持坐标系平移，方便与其它测量系统对齐。

2. 流场物理量计算

基于速度场 (u, v) 自动计算：

  • 基础量：合速度、涡量、散度、速度梯度分量 $ |\nabla u|, |\nabla v| $
  • Liutex：严格遵循刘超群教授的公式，输出纯旋转强度 \(R\)、剪切贡献 \(S\) 以及局部旋转角速度 $\lambda_{ci}$

计算核心完全与界面分离，可单独调用，便于批处理或嵌入其它脚本。

3. 可视化与交互

  • 使用 Matplotlib 嵌入 PyQt6 界面，利用 轮廓填色（contourf） 显示标量场，并可叠加 速度矢量（quiver） 和 流线（streamline）。
  • PLIF 叠加：将处理后的 PLIF 标量场以半透明形式覆盖在速度场之上，直观显示标量与流动结构的耦合。
  • 实时动画：支持播放/暂停、拖动滑块浏览序列帧。
  • 交互工具栏：Matplotlib 内置导航工具栏，支持缩放、平移、保存图片等操作。
  • 可随时将当前显示的标量场数据导出为 .npy 文件，便于后续分析。

4. 单点 FFT 分析

在流场图上点击任意位置，软件自动提取该点所有时间步的物理量（如 Liutex、涡量）序列，并进行 FFT 变换，绘制单边幅值谱。这对于识别流动中的周期性特征（如尾涡脱落频率）非常实用。

气泡雾化喷嘴实验

气泡雾化喷嘴通气/不通气对比实验

一、为什么要做这个对比实验？

在之前的组会汇报和博客里，我们反复讨论过气泡雾化喷嘴的核心原理——雾化的动力不是液体的高压，而是气泡在喷口处的爆破。 这个概念用文字说一百遍，不如两张对比图来得直观。所以，我用简化后的三通+金属泡沫喷嘴原型，做了一组简单的“不通气 vs 通气”对比实验，用同一套供液条件，直观展示气体在气泡雾化中扮演的关键角色。

二、实验设置

  • 喷嘴：三通 + 金属泡沫 + 螺帽（排放孔 φ1.0mm）简化原型
  • 液体：水
  • 供液：齿轮泵（后续已计划更换为柱塞泵）
  • 供气：氮气瓶 + 减压阀，气体压力略高于液体压力
  • 拍摄：手机高速摄影模式 两组对照条件： | 工况 | 液体压力 | 气体供给 | 备注 | | 不通气 | 泵正常供液 | 关闭气体阀门 | 纯压力雾化模式 | | 通气 | 泵正常供液 | 打开气体阀门（低压，~0.4 MPa） | 气泡雾化模式 |

三、实验结果

3.1 不通气状态

在关闭气体阀门后，液体仅依靠齿轮泵的压力通过喷嘴。可以看到：

  • 液体呈连续液柱或粗大液丝流出，几乎看不到雾化效果。
  • 液体在离开喷口后，靠自身重力和初始动量下落，表面张力使其保持完整。
  • 喷口附近没有任何“爆破”或分散的迹象。 一句话总结：没有气体，这就是个“滋水枪”，不是“喷雾器”。

3.2 通气状态

打开气体阀门后，压缩氮气通过金属泡沫微孔注入液体，形成气液两相泡状流。可以看到：

  • 喷口处立刻形成明显的雾化锥，液体被分散成大量细小液滴。
  • 喷雾呈扇形向外扩散，肉眼可见的液滴群代替了原先的连续液柱。
  • 即便液体压力和流量没有改变，雾化效果发生了质的变化。 一句话总结：气体一打开，水柱变雾锥——这就是气泡爆破的力量。

四、现象背后的原理

这组对比实验完美诠释了气泡雾化的核心机制： 1.不通气时（纯压力雾化）：液体雾化依赖高压力产生的剪切力。齿轮泵提供的压力远不足以让水在1.0mm小孔中达到雾化所需的速度，所以只能形成液柱。 2.通气时（气泡雾化）：气体在混合室（三通 + 金属泡沫）内与液体混合，形成大量被液膜包裹的微小气泡。当这些气泡随液体一起从排放孔喷出时，外部压力骤降，气泡急剧膨胀并爆破，把包裹它的液膜“炸”成微小液滴。这个过程的驱动力是气泡的内外压差，与液体压力大小关系不大。 这也正是金如山老师在文献笔记里反复强调的——“气泡雾化不依赖液体高压，靠气泡爆破”。

火焰探测器原理及使用

为了更好的观察火焰的吹熄，购买了型号为YJH-1E的紫外火焰探测器，如图。

紫外火焰探测器分为传感器和检测器两部分。

传感器核心是紫外光电管，这是一种特殊的气体放电管。其外壳采用紫外透射玻璃，仅允许200-290nm波长的紫外线通过，同时阻挡可见光和大部分红外线，具备“日盲”特性。 电极系统由阴极和阳极组成：阴极为碲化铯（Cs-Te）材料，这是一种化合物半导体光电发射材料，具有低逸出功且对可见光不敏感的特性，受紫外光子照射时易发射光电子；阳极为金属丝结构，工作时施加约350V高压偏置，在阴阳两极间形成强电场吸引电子。管内填充低压惰性气体（如氖气、氩气或其混合物），气压在几毫巴到数十毫巴量级，低压条件使气体分子更容易被加速电子撞击而发生电离。 当火焰发出的紫外光穿过玻璃窗照射到阴极表面时，紫外光子能量打出阴极表面的光电子。光电子在强电场中被高速加速，获得足够动能后撞击气体分子，使其电离释放出更多电子和新离子；新生电子继续被电场加速并撞击更多气体分子，形成链式倍增——这一过程像“雪崩”一样迅速放大，称为气体倍增效应。当电子数量累积到一定程度时，管内发生瞬态放电，瞬间有较大电流通过阴阳两极。此时，探测器内部的淬灭电路会快速降低阳极电压，使放电立即终止，管子“复位”准备探测下一个光子。每次雪崩放电与随后的淬灭动作，就在外部负载电阻上完成一个完整的“探测-复位”循环，并形成一个微弱的电流脉冲。此时，光电管本身输出的微弱电流脉冲无法被普通采集设备直接读取，因此传感器模块内部设有负载电阻，在输出端产生一个可被读取的电压脉冲——这个电压脉冲即为进入检测器的原始信号。 传感器通过两根引线与探测器连接：棕线和蓝线。其中棕线通常接阳极（正极），蓝线接阴极（负极）。这两根线承担双重功能——探测器通过它们向传感器提供工作高压，同时也通过它们接收传感器输出的脉冲信号。 检测器的作用是给传感器供电提供偏置电压，并将传感器传来的脉冲电压信号进行电路处理，进而转化为4-20mA电流信号。检测器上设有"DC24V+激励电压输入"端子，内部设有DC-DC升压电路，将24V升压至约350V，再通过棕线和蓝线施加在紫外光电管的两端，形成强电场。电路处理则是对传来的信号进行脉冲整形与单片机计数，最终经PWM驱动输出为4-20mA标准电流信号。 将火焰探测器连接4-20mA转rs485采集模块并通过usb用电脑读数。设备接线如图。 刚开始采集模块数据的读存是用其配套的软件来完成。进行实验后发现，自带的软件仅支持一秒五次的数据读储，无法很好的捕捉吹熄的细节。 实验过后，发现它的采集频率为30Hz，即采集模块的硬件是可以支持更高频的读存的，但是自带软件却无法完成，通过python写一段串口读存程序，可以提高读存的数量。

# ================== 串口初始化 ==================
instrument = minimalmodbus.Instrument(PORT, SLAVE_ADDRESS)
instrument.serial.baudrate = BAUDRATE
instrument.serial.timeout = 0.8
instrument.serial.write_timeout = 0.8
instrument.serial.parity = minimalmodbus.serial.PARITY_NONE
instrument.serial.stopbits = 1
instrument.serial.bytesize = 8
instrument.mode = minimalmodbus.MODE_RTU
instrument.clear_buffers_before_each_transaction = True

# 解析函数
def get_ma(raw):
    s = str(raw).zfill(5)
    leading = int(s[0])
    mantissa = int(s[1:])
    voltage = mantissa * (10 ** -leading)
    return round(voltage / 249 * 1000, 3)

# ================== 读取函数 ==================
def read_safe():
    for _ in range(2):
        try:
            return instrument.read_register(CHOSEN_CHANNEL, functioncode=FUNCTION_CODE, signed=False)
        except:
            time.sleep(0.002)
    return None

# ================== 主采集循环 ==================
target_time = start_time + sample_count * SAMPLE_INTERVAL
while time.perf_counter() < target_time:
    pass
sample_count += 1

raw = read_safe()
if raw is None:
    continue

ma = get_ma(raw)
timestamp_str = datetime.now().strftime('%Y-%m-%d %H:%M:%S.%f')[:-3]

data_batch.append({
    '时间': timestamp_str,
    '通道3_mA': ma,
    'raw': raw
